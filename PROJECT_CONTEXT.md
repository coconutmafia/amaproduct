## Что это за продукт

**AMA (amaproduct.com)** — AI-сервис для блогеров/экспертов: «личный продюсер + мозг контента».
Не генератор общего назначения, а система, которая впитывает голос конкретного человека
(Tone of Voice, его посты, кастдев, карту смыслов, линии блога, анализ его и конкурентов
Instagram) и пишет контент ИМЕННО как он, под методологию прогрева/запуска.

Владелец — Августа Василик, продюсер микроблогеров. Её клиенты приходят из основного
продукта (продюсирование) и нативно переводятся в сервис.

## Стек / архитектура

- Next.js 16 (App Router, Turbopack) + React 19, TypeScript
- Supabase (Postgres + Auth + Storage + RLS)
- Anthropic Claude (генерация) — `lib/ai/client.ts`, `MODEL`
- OpenAI Whisper (расшифровка аудио), Apify (скрейп Instagram)
- Хостинг Vercel **Pro** (maxDuration 300s — важно для длинных AI-запросов)
- Деплой: push в `main` → авто-деплой. Коммит-формат с `Co-Authored-By`.
- **Миграции БД применяет владелец вручную** (SQL Editor в Supabase) — у ассистента нет DDL-доступа.
  Каждую новую миграцию давать готовым SQL-блоком.

### Ключевые модули
- `lib/ai/rag.ts` — `buildRAGContext`: подтягивает материалы проекта в генерацию.
  ВАЖНО: `ALWAYS_INCLUDE` (tone_of_voice, my_instagram, competitors, meanings_map,
  unpacking_map, blog_lines, audience_research) подтягивается ВСЕГДА, в обеих ветках.
- `lib/ai/prompts/system.ts` — системный промпт (голос, форматирование, запрет хэштегов).
- `lib/ai/prompts/content-brain.ts` — хуки, фазы, схемы, `getViralReelsFramework`.
- `lib/supabase/upsertMaterial.ts` — ручной upsert материалов (нет unique-констрейнта на triple).
- `lib/completeness.ts` — единая формула «полноты базы».

## Что СДЕЛАНО (ключевое)

- Исследование аудитории: загрузка аудио → Whisper (через Supabase Storage, обход лимита Vercel)
  → таблица интервью → карта смыслов (CSV-выгрузка с группировкой).
- Материалы проекта: ToV (в т.ч. «из моих текстов» → сохраняется как style_examples),
  распаковка, карта смыслов, линии блога, кейсы, **Instagram (свой + до 5 конкурентов через Apify)**,
  импорт из другого проекта (починен — писал не в те колонки).
- План прогрева (warmup) + AI-правка (в т.ч. на черновике в мастере, шаг 8).
- Контент-план: двухшаговый (сначала «Сгенерировать план» брифов → потом контент).
  Генерация единицы открывает **чат-ассистента** с темой дня (`?prompt=&back=content-plan`).
  Разные форматы дня = разные углы одной темы. Брифы не выдумывают фейковые кейсы.
- **AI-ассистент** (`/projects/[id]/assistant`) — чат, знает проект, пишет голосом.
- **Standalone «Создать»** (`/create`) — генерация без проекта, на методологии (для тестов гипотез/ниш).
- Виральный движок рилз. Хэштеги убраны везде. StructuredContentView (читаемые рилз/карусели/сторис).
- **Календарь** (`/projects/[id]/calendar`) — визуальный месяц.
- **Результаты** (`/projects/[id]/results`) — внести охваты/реакции → топ-посты становятся эталоном (замкнутый цикл).
- **Тренды месяца** (`/admin/trends`) — владелец добавляет актуалочки → вплетаются в контент-планы.
- **Админ-аналитика** (`/admin/analytics`) — по каждому юзеру: проекты/материалы/контент/генерации/оценка
  стоимости API + кнопка «Триал 2 мес» (ставит `subscription_expires_at`).
- Возврат квоты при ошибке генерации. Единая «полнота базы». ContentEditor синкает сгенерированный текст.
- Голосовой ввод: рабочий паттерн (continuous + interim + автоперезапуск) в VoiceTextarea и AiEditChat.

   Сейчас счётчик генераций есть, платных планов нет. После — гейтинг лимитов по тарифу.

   - Карусели → реальные слайды-картинки через `@vercel/og` (HTML→PNG, без внешних API). НАЧАТЬ С ЭТОГО.

   - Стиль ИНДИВИДУАЛЬНЫЙ на каждый проект (цвета/шрифт/лого), авто-подсказка из Instagram.
   - Ждём от владельца РЕФЕРЕНСЫ каруселей (стиль) перед стартом.
3. **Автопостинг** — отложен (нет смысла без визуала). Telegram: 1 общий бот, юзер добавляет

4. **Командная работа** (продюсер приглашает в проект) — нужна таблица project_members + аккуратная RLS.
5. **Многоязычность** — в самом конце, перед мировым запуском (большая механическая работа).
6. **Мобильное приложение** — после стабилизации веба, нативное/React Native (чтобы диктовка работала нативно).

## Важные решения / ограничения (не переоткрывать)

- **Диктовка на iPhone (Safari)**: текст появляется фразами после паузы — это ограничение Web Speech API
  Apple, в вебе не чинится. Нативно (в приложении) будет ок. Везде показываем подсказку про это.
- **Vercel build «слетает» иногда** = транзиентный сбой сборщика Next 16, не код. Решение: пустой коммит / redeploy.
  Если tsc локально чистый и `npx next build` проходит — код в порядке.
- **Локальная сборка**: на macOS нужен `cp node_modules/lightningcss-darwin-x64/lightningcss.darwin-x64.node node_modules/lightningcss/` перед `npx next build`.
- **Всегда `npx tsc --noEmit` ДО коммита** (несколько раз ловили TS-ошибки уже в билде Vercel).
- Карта смыслов / большие AI-ответы: всегда нормализовать вывод модели (customer_words мог прийти строкой),
  сканировать ВСЕ текстовые блоки (не content[0]), использовать tool_use для гарантированного JSON где можно.

## Стратегия (для контекста)

- Тарифы (рекомендация, не вшиты): Solo $50 / Pro $99 / Продюсер $199-249. Главный сегмент — продюсеры (мульти-проект).
- Триал: ~50 клиентов из основного продукта, 2 месяца бесплатно (механизм готов в админ-аналитике).
- Дифференциация: голос + методология + кастдев + анализ конкурентов — этого нет у Taplio/Hypefury/Predis и рус-агрегаторов.
