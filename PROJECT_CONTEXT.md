## Что это за продукт

**AMA (amaproduct.com)** — AI-сервис для блогеров/экспертов: «личный продюсер + мозг контента».
Не генератор общего назначения, а система, которая впитывает голос конкретного человека
(Tone of Voice, его посты, кастдев, карту смыслов, линии блога, анализ его и конкурентов
Instagram) и пишет контент ИМЕННО как он, под методологию прогрева/запуска.

Владелец — Августа Василик, продюсер микроблогеров. Клиенты приходят из основного продукта
(продюсирование) и нативно переводятся в сервис. ~50 клиентов на бесплатный триал 2 мес.

### Как тестировать вживую (воспроизводимо — этим пользовались в июне 2026)
Фичи за авторизацией/RLS проверяются так (НЕ просто tsc/build):
1. **Тест-аккаунт сервис-ключом** (node + `@supabase/supabase-js`, env из `.env.local`): создать confirmed-юзера
   `auth.admin.createUser({email,password,email_confirm:true})`, засидить проект/материалы/`warmup_plan` (status
   'approved', `plan_data.warmup_plan.phases[].daily_plan[]` с `briefs`). Креды: `ama-qa-bot@gmail.com` / `QaTest!23456`.
   Очистка СТРОГО по порядку (FK): content_items→warmup_plans→project_materials→projects→saved_content→content_trends→
   profiles, ПОТОМ `auth.admin.deleteUser` (иначе «Database error deleting user» из-за FK профиля). `listUsers` —
   пагинируй (perPage 200, до ~20 стр.), иначе юзера не находит.
2. **Дев-сервер через Claude Preview MCP**: `.claude/launch.json` = `{name:"ama-dev", runtimeExecutable:"npm", runtimeArgs:["run","dev"], port:3000}` → `preview_start`. Мобильный вид: `preview_resize preset:"mobile"`.
3. **Логин в браузере**: `preview_fill`/`preview_click` НЕ обновляют React-стейт надёжно. Рабочий способ — `preview_eval`:
   нативный сеттер `value` + dispatch `input`, затем `form.requestSubmit()`. Навигация — `window.location.href` (иногда
   нужен 2-й раз после старта сервера). Клики — `preview_eval` (найти кнопку по тексту + `.click()`). Замеры — `getBoundingClientRect`.
4. **Убрать за собой**: удалить тест-юзера+данные, `.claude/launch.json`, временные `qa-temp.mjs`/тест-скрипты, `.next`.
DDL ассистент применить НЕ может — даёт SQL, владелец применяет в SQL Editor.

### Ключевые модули
- `lib/ai/rag.ts` `buildRAGContext` — подтягивает материалы в генерацию. `ALWAYS_INCLUDE` = ВСЕ
  существенные типы материалов (голос, кейсы, воронка, стратегия, тактика, конкуренты, кастдев,
  карта смыслов, распаковка, линии, Instagram) подтягиваются ВСЕГДА (slice 3000). Используют:
  `/api/ai/generate`, `/api/ai/edit`, `/api/ai/chat`.
- `lib/ai/prompts/system.ts` — системный промпт (голос/словечки/форматирование/без хэштегов).
  Анти-AI-тэллы: `AI_TELLS_TO_AVOID` в `content-brain.ts` (тире «—» НЕ ставить — главный палящий признак;
  без телеграфных обрывков «Не А. Не Б. А В.»; офферы конкретные, не «скажу конкретно»/«разбираю внутри»).
  Вшит во ВСЕ генераторы: chat (через buildSystemPrompt + standalone), generate, edit, и в валидатор постов.
- `lib/ai/prompts/content-brain.ts` — хуки, фазы, схемы, `getViralReelsFramework`.
- `lib/supabase/upsertMaterial.ts` — ручной upsert материалов (нет unique-констрейнта на triple).
- `lib/completeness.ts` — единая формула «полноты базы». `lib/cleanText.ts` — `cleanMarkdown`.

## Что СДЕЛАНО

- Исследование аудитории: аудио→Whisper (через Supabase Storage, обход лимита Vercel)→таблица→карта смыслов (CSV).
- Материалы: ToV (+«из моих текстов»→сохраняется как style_examples), распаковка, карта смыслов, линии,
  кейсы, воронка, стратегия, тактика, **Instagram (свой+5 конкурентов через Apify)**, импорт из др. проекта.
- План прогрева + AI-правка (вкл. черновик в мастере, шаг 8; правит и `meaning`, и брифы форматов).
- Контент-план: двухшаговый. Шаг 1 — «Сгенерировать план» (AI расписывает брифы под форматы дня).
  Шаг 2 — кнопка «Сгенерировать рилз/сторис/пост» открывает **чат-окно** (`/projects/[id]/assistant?gen=1&day=&type=&phase=&brief=&back=content-plan&week=N`):
  сверху сообщение AI с темой дня + формат-подсказкой (сторис коротко, рилз по секундам), оно ЖДЁТ ввода
  (не авто-генерит) → юзер дописывает детали → AI пишет юнит в чате (правится как в ChatGPT). На ответе:
  «Сохранить в Готовое» (библиотека) + «В план» → `/api/content-items` сохраняет как content_item дня (✓).
  Назад — кнопка «← Контент-план» (backHref с `?week=N`, страница плана читает `?week`), возвращает на ТУ неделю.
  Чистый вывод: send() шлёт `genFormat` → chat-роут включает «режим генерации» (только сам текст, без вводных
  «Окей/Вот/Держи»), чтобы в сохранённое не попадала болтовня AI. Опенер (`opener:true`) — без кнопок сохранения.
  Уже сгенерированное в плане: «Посмотреть»/«Обновить» (Обновить = снова в чат).
  ВАЖНО (история, не переоткрывать вслепую): инлайн-панель деталей была (июнь 2026), но владелец попросила
  заменить на чат-окно — генерить инлайн в плане ей неудобно. `/api/ai/generate` (структурный) больше НЕ
  зовётся из контент-плана (остался для стр. `/generator`).
- **AI-ассистент** (`/projects/[id]/assistant`) — чат, знает проект, пишет голосом.
- **«Создать»** (`/create`) — чат, есть селектор проекта (по умолчанию — последний проект → полный контекст;
  «без проекта» → только методология). Нижняя навигация «Создать» ведёт сюда.
- Виральный движок рилз. Хэштеги убраны везде. StructuredContentView (рилз/карусели/сторис читаемо).
- **Календарь** (`/projects/[id]/calendar`), **Результаты** (`/projects/[id]/results` — замкнутый цикл:
  внёс охваты → топ-посты становятся эталоном стиля).
- **Тренды месяца** (`/admin/trends`) + **Виральные рилз-референсы** (`/admin/viral-reels` системные и
  `/projects/[id]/viral-reels` пользовательские) → вплетаются в контент-планы по нише. Бэкенд `/api/viral-reels`
  (Apify скрейп рилз → Whisper расшифровка → AI-разбор → сохранение).
- **Админ-аналитика** (`/admin/analytics`) — по юзерам + кнопка «Триал 2 мес».
- Голос везде через **запись→Whisper** (логика в хуке `lib/useVoiceRecorder.ts`). В чатах (`/create`,
  ассистент) — общий `components/ui/ChatComposer.tsx`: запись разворачивается в **панель во всю ширину**
  (отмена · живая волна + таймер · подтвердить), как «Release to send» у ChatGPT. В формах — компактная
  `components/ui/VoiceRecordButton`. Web Speech не работал в webview/iOS — поэтому только запись→Whisper.
- Чат: читается сверху вниз. Логика пина — хук `lib/useChatPin.ts`: вопрос пинится к верху явным `scrollTo`
  (rect-based, НЕ scrollIntoView — тот промахивался на мобиле) + **динамический** нижний спейсер (заполняет
  пустоту под коротким ответом, схлопывается в 0 под длинным). КЛЮЧЕВОЕ: `lastUserRef` вешается на ПОСЛЕДНЕЕ
  сообщение юзера через `lastIndexOf('user')` (а не «последнее в массиве» — иначе реф отваливался, как только
  приходил ответ ассистента, и вопрос переставал держаться сверху). Без markdown-символов.
- **Вёрстка фулл-скрин чатов (НЕ ломать):** корни `/create` и `/assistant` — `h-full` (НЕ `100vh`!), и
  `components/shared/PageTransition.tsx` — `h-full` (НЕ `min-h-full`). Иначе ломается цепочка высот → `<main>`
  двойной скролл → поле ввода уезжает под нижнюю навигацию (юзер не находит). Обычные страницы скроллятся как
  раньше (контент перерастает `<main>`). Шапка контент-плана — `sticky top-0`, чтобы «AI-правка» не уезжала.
- Чат-роут `/api/ai/chat`: maxDuration 300, max_tokens 8000 + авто-дописывание при stop_reason=max_tokens
  (цикл до 4 раундов) — чинит обрыв «5 рилзов» на 2-м (раньше резал 60-сек лимит функции).
- Выгрузка текстовых материалов → стилизованный HTML (нормальный шрифт).
- **Библиотека «Готовое»** (`/library`, таб в нижней навигации + сайдбаре): одобренный контент сохраняется
  в папку, сгруппирован по типам, с копированием/удалением. Кнопки «Копировать»+«Сохранить» — СВЕРХУ каждого
  ответа ассистента (в `/create` и ассистенте) и в просмотрщике контент-плана. Хелперы: `lib/saveContent.ts`,
  `lib/contentToText.ts` (флэттенит структурный контент в текст), `components/content/SaveButton.tsx`.
  Таблица `saved_content` — см. «Миграции».
